Architecture & Flow

End-to-end implementation map inferred from source code.

1) CLI Entrypoint

`neuromap` dispatches `train`, `predict`, and `serve` subcommands.

2) Training Pipeline

MONAI transforms + patch sampling + 3D U-Net + DiceCE loss + validation Dice + checkpoint history.

3) MC-Dropout Inference

Dropout layers stay active during inference to emit mean probability, variance, and entropy.

4) FastAPI Slice Service

Cases are registered from local NIfTI paths, then served as per-slice PNG and optional 3D scatter payload.

5) Next.js Interactive Viewer

Frontend includes health checks, case registration, overlays, slice metrics, and Plotly 3D point cloud.

6) API Bridge Proxy

Same-origin route proxies requests and supports `x-neuromap-api-url` for configurable backend target.

Insight Matrix

Actionable findings from implementation review.

High

Unrestricted file-path ingestion in case registration

API accepts raw local paths (`image_path`, `prediction_path`, `entropy_path`) from request body and loads with nibabel. This is unsafe for exposed environments.

Evidence: backend/neuromap/api.py (register + _load_volume)

High

Open CORS and no auth controls

CORS allows all origins/methods/headers and there is no authentication on data-serving endpoints.

Evidence: backend/neuromap/api.py (CORSMiddleware config)

High

Proxy target can be overridden via request header

Bridge route reads `x-neuromap-api-url` from caller and forwards upstream. If deployed broadly, this can create SSRF-style risk.

Evidence: frontend/app/api/bridge/[...path]/route.ts

Medium

In-memory case registry has no eviction/persistence

Registered volumes stay in global process memory (`CASES`) with no TTL, size guard, or external storage.

Evidence: backend/neuromap/api.py (global CASES dict)

Medium

Packaging configuration mismatch risk

`pyproject.toml` points setuptools packages to `src`, but repository does not include `src` directory in current tree.

Evidence: pyproject.toml + repository layout

Medium

No automated tests found

Project-owned code has no obvious test suite, which raises regression risk across training, inference, and API behavior.

Evidence: repository search for test/spec files

Low

Strength: robust MC-dropout implementation

Dropout activation is explicit and uncertainty outputs include mean probability, variance, and predictive entropy.

Evidence: backend/neuromap/uq.py

Low

Strength: practical Windows stability defaults

Training defaults to conservative DataLoader/cache settings on Windows and prints explicit guidance.

Evidence: backend/neuromap/train.py
